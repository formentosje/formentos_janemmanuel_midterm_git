# Midterm Examination Reflection

1. What is Git used for?
   Git is a version control system that is used to track changes in files and collaborate with multiple developers on a project.

2. What is the difference between Git and GitHub?
   Git is a local version control tool that runs on your machine, while GitHub is a cloud-based platform that hosts Git repositories online. GitHub adds features like pull requests, issue tracking, and remote collaboration on top of Git.

3. Why is branching useful?
   Branching allows developers to create isolated environments to work on features or fixes without affecting the main codebase. It enables parallel development and makes it easier to test changes before merging them into the main branch.

4. What command did you use to upload your local commits to GitHub?
   I used the `git push` command to upload my local commits to the remote GitHub repository. This command sends the committed changes from local branch to the matching branch on GitHub.

5. What did you learn from this activity?
   I learned how to initialize a Git repository, stage and commit changes, and push them to a remote GitHub repository. This activity also reminded me of the importance of version control for collaborative and organized software development.
